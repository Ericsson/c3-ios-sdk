Conference

public final class Conference: EventEmitter

A WebRTC call between multiple peers.

The conference only manages the connections to a set of peers. Functionality is added by attaching different RtcComponents using Conference.attach(_:as:).

Version

1.0.0

Events

  • error: Emitted when there is an error within the conference, e.g. the signaling fails.
  • isClosed

    True if the conference has been closed.

    A closed conference can not be started again, and will have had all attachments removed. It will alse have been removed from Room.conference, so that a new conference can be started in the room.

    True if the conference is stopped. This is the initial state for incoming conferences.

    Stopped conferences will keep all attachments and state, but will not have any connections to other peers.

  • attach(_:as:)

    Attaches a component to the conference. If the peer attaches a compatible component to the same attachment point, the two components will be paired together.

    A common use case is for two peers to each attach a data share to "data“. Once the conference has connected, both peers will be able to use the attached data share to share data.

    The attachment point name can be any string, although excessively long names should be avoided.

    This method can be called at any point during a call, but will have no effect if the conference is closed.

  • detach(_:from:)

    Detaches the component currently attached to the given attachment point. If no component is currently attached, an error will be thrown. If the component argument is given and it does not match the currently attached component, an error will be thrown.

    This method can be called at any point during a call, but will have no effect if the conference is closed.
