RtcPeer

final public class RtcPeer: MediaNode

This class provides a point of interaction with remote peers of an attached component. Peers are accessed either through AttachPoint.peers, or via the overridable methods of an attach point.

  • sendMessage(_:)

    Sends a message to the peer. The message will always be received by the peer in the same order as they are sent.

    If the connection to the peer is lost, the messages will be resent when the peer reconnects. This means that excessively large messages should be avoided, since if a peer does not reconnect, the messages will the stored in memory until the component is detached.

    In case the above functionality isn’t desired or useful, it is better to use data channels instead, using RtcPeer.createChannel(_:configuration:).
