  • receivedPeer(_:)

    This method is called whenever a peer is added to the AttachPoint.peers collection of the attach point.

    Only peers to who we have an active connection and who have mounted a matching component will be visible.

    This method will always be followed by a call to RtcComponent.lostPeer(_:) for the given peer. It will also only ever be called while the component is attached.

  • lostPeer(_:)

    This method is called whenever a peer is removed from the AttachPoint.peers collection of the attach point.

    This will happen when the connection to a peer is lost, the peer detaches their component, or when the local component is detached.

    This method will always preceeded by a call to RtcComponent.receivedPeer(_:) for the given peer. It will also only ever be called while the component is attached.

  • receivedChannel(_:peer:)

    This method is called whenever a peer opens a data channel. The channel will be ready to use as soon as this method is called.

    When this method is called the channel will already have been added to the collection returned by RtcPeer.remoteChannels property.

    This method will only be called for active peers, which means that it is always happens after the RtcComponent.receivedPeer(_:) call for a peer, and before RtcComponent.lostPeer(_:). It will also only ever be called while the component is attached.

  • willAttach(_:)

    A overridable method that is called when the component is about to be attached to something. When this method is called, RtcComponent.attachPoint will not yet have been set, but the attachment point can be accessed through the parameter.

    If this method throws an error, the component will fail to attach. The error will be caught and handled by logging an error, but the program flow will continue.

    This method will always the followed by a call to RtcComponent.willDetach().

    Version

    1.0.0

    Declaration

    Swift

    open func willAttach(_ attachPoint: AttachPoint)

    Parameters

    attachPoint

    The point that the component will be attached to.
  • willDetach()

    Called before the component is detached. Once this method has returned the attachPoint will no longer be valid, and any attempt to call a method on the attach point will throw an error.

    This method will always the preceeded by a call to RtcComponent.willAttach(_:).
