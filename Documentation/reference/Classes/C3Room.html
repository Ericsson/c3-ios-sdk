C3Room

final public class C3Room: C3EventEmitter

A representation of a room that users join in order to communicate.

Version

1.0.0

Events

  • event:<type>: Same as event, but listens to a specific event type. Receives C3Event instance.
  • avatar: Emitted when the avatar of the room is changed. Receives C3ImageResource instance or nil.
  • call: Emitted when there is an incoming call to this room. Incoming calls are stopped and need to be started with call’s start() before a connection is set up. Receives C3Call instance.
  • event: Emitted when a new event is triggered in the room. Will not trigger when loading old events. Receives C3Event instance.
  • events: Emitted when the list of events in the room change. This event does not have any parameters, the updated list of events can instead be accessed via the events property.
  • guestAccessRule: Emitted when the guest access rule of the room changes. Receives C3GuestAccessRule instance.
  • joinRule: Emitted when the join rule of the room changes. Receives C3JoinRule instance.
  • members: Emitted when the members user collection is changed. This event does not have any parameters, the updated list of users can instead be accessed via the members property.
  • membership: Emitted when the authenticated users membership of the room is changed. Receives C3RoomMembership instance.
  • name: Emitted when the name of the room is changed. Receives the new name.
  • otherMembers: Emitted when the otherMembers user collection is changed. This event does not have any parameters, the updated list of users can instead be accessed via the otherMembers property.
  • topic: Emitted when the topic of the room is changed. Receives the new topic.
  • typing: Emitted when the list of users who are currently typing is changed. This event does not have any parameters, the updated list of users can instead be accessed via the typing property.

  • load(count:chunkSize:filterFunction:success:failure:)

    Load in old events. If multiple requests are sent at once they will be queued after eachother.

    By passing a filterFunction, it is possible to filter whether each event should count towards the total count, and keep loading in events until the total count has been reached. Events that are filtered out by the filter function are not discarded, they simply don’t count towards the load count.

    If a filter function is used the number of matching events might exceed the requested count, since events are loaded in chunks and no events will be ignored.

    Version

    1.0.0

    Parameters

    chunkSize

  • setTyping(_:timeout:margin:success:failure:)

    Sets whether the user is currently typing or not, with an optional timeout and margin. The timeout can be used to temporarily set typing to true, while the margin allows throttling of requests.

    The margin permits setTyping to be called at a high rate without actually sending any requests to the server. A request is sent only if the end of the previous request’s timeout has been reached or if the time until then is smaller than the margin.

    e.i. if (current time + margin) > (previous (request time + timeout))

    Specifying a timeout is only valid when setting typing to true, as setting it to false will always do so indefinetly.

    Setting the typing state will cause any previous state, including timeouts, to be overwritten.

    Version

    1.0.0

    Declaration

    public func setTyping(
        _ typing: Bool = true,
        timeout: TimeInterval = 10.0,
        margin: TimeInterval = 1.0,
        success: C3SuccessCallback<C3Room>? = nil,
        failure: C3FailureCallback? = nil)

    Parameters

    typing

    The new typing state.
    timeout

    margin
