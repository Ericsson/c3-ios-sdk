PowerLevelsEdit

final public class PowerLevelsEdit: PowerLevelsContainer

A class which provides a fluent API for modifying a power levels object. It cannot be instantiated directly and can only be created through the static factory methods as well as Room.editPowerLevels().

Many methods are verbs, but they simply set the power level required to execute the matching action, with the exception of PowerLevelsEdit.commit.

No actual action is taken until the PowerLevelsEdit.commit method is called. The commit method is only available if the instance was obtained from a Room.editPowerLevels() call. It is also possible to create an instance with one of the static factory methods and then pass the instance to Room.setPowerLevels(_:success:failure:).

Version

1.0.0

Example

room.editPowerLevels()
    .kick(has: 60)
    .addState(has: 0)
    .user(promotedUser, has: 60)
    .event(my.customer.event, has: 20)
    .commit({
        print("Power levels updated!)
})



room.setPowerLevels(
    PowerLevelsEdit.fromDefault(client.user!),
    success: { room in
        print(Power levels reset)
})

  • commit

    Commits the current state of the power levels edit by updating the power levels of the room to the current values. This method is only present if the instance was obtained via Room.editPowerLevels().

    Note

    Power levels can no longer be edited after successful commit. In order to perform further changes, Room.editPowerLevels() has to be called again.
